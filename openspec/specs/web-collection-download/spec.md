## Requirements
### Requirement: 取り込み先コレクションの選択

ダウンロードダイアログは、汎用Web記事の取り込み先として「新規コレクション（名前を入力）」または「既存コレクションに追加」をユーザが選択 SHALL できる。新規コレクションの既定名は抽出した記事タイトル MUST とする。既存コレクションの選択肢は `siteType='web'` のコレクションに限定 MUST する。

#### Scenario: 新規コレクションを作成して取り込む
- **WHEN** ユーザが記事URLを入力し「新規コレクション」を選んでコレクション名を確定する
- **THEN** システムは新しい `siteType='web'` のコレクションフォルダを作成し、その記事を最初のエピソードとして保存する

#### Scenario: 新規コレクション名の既定値は記事タイトル
- **WHEN** ユーザが記事URLを入力し「新規コレクション」を選ぶ
- **THEN** システムはコレクション名の初期値として抽出した記事タイトルを提示し、ユーザは編集できる

#### Scenario: 既存コレクションに追加する
- **WHEN** ユーザが記事URLを入力し「既存コレクションに追加」で既存の web コレクションを選ぶ
- **THEN** システムはそのコレクションフォルダへ記事をエピソードとして追記する

#### Scenario: 既存コレクション選択肢は web に限定
- **WHEN** ユーザが「既存コレクションに追加」の選択肢を開く
- **THEN** システムは `siteType='web'` のコレクションのみを候補として表示し、専用サイトの作品は表示しない

### Requirement: 空コレクションの事前作成

ライブラリは、記事を1件も持たない空のコレクションを事前に作成 SHALL できる。作成された空コレクションは「既存コレクションに追加」の候補に現れる MUST。

#### Scenario: 空コレクションを作ってから記事を集める
- **WHEN** ユーザがライブラリで空コレクションを名前付きで作成する
- **THEN** システムは記事0件の `siteType='web'` コレクションを作り、以降のダウンロードで「既存コレクションに追加」の候補として選べる

### Requirement: エピソードの追記と採番

システムは、記事をコレクションへ追記する際、既存エピソードの最大インデックス + 1 を採番して末尾に追加 SHALL する。記事タイトルが当該エピソードのタイトルになる MUST。エピソードファイル名のゼロ埋め幅は固定とし、追記による既存ファイルのリネームを発生させてはならない MUST NOT。

#### Scenario: 2件目以降を末尾に追記する
- **WHEN** 既に1件のエピソードを持つコレクションに新しい記事を追加する
- **THEN** システムはインデックス2として末尾にエピソードを追加し、既存エピソードのファイル名は変更しない

#### Scenario: 追記後も下流機能がそのまま効く
- **WHEN** コレクションに複数の記事エピソードが保存されている
- **THEN** LLM解析・TTS・検索・閲覧はそのコレクションフォルダを既存の複数エピソード作品と同様に扱い、無改修で動作する

### Requirement: 同一URL再取得時の更新（重複追加しない）

システムは、コレクションへの追記時に記事の同一性キーを URL とし、フォルダ内の `episode_cache`（`url` 主キー）で管理 SHALL する。既に同一コレクションに追加済みの URL を再ダウンロードした場合、対応するエピソードを上書き更新 MUST し、新規エピソードとして重複追加してはならない MUST NOT。別コレクションへの同一URLは独立したエピソードとして扱う MUST。

#### Scenario: 同一URLの再取得はエピソードを更新する
- **WHEN** ユーザが、あるコレクションに既に追加済みの URL を同じコレクションへ再度ダウンロードする
- **THEN** システムは `episode_cache` で該当エピソードを特定し、その内容を上書き更新して重複エピソードを作らない

#### Scenario: 別コレクションへの同一URLは別エピソード
- **WHEN** ユーザが同じ URL を別のコレクションへ追加する
- **THEN** システムはそのコレクション内では新規エピソードとして保存し、各コレクションが独立した `episode_cache` を持つ
