## Purpose

ライブラリ内の小説ファイルから「実効エピソード番号」（ファイルの数値プレフィクス、無ければフォルダ内の 1 始まり辞書順位）を導出する単一の共有規則を提供する。要約トリガのスコープ上限解決・要約結果のスコープフィルタ・v5 migration の `coveredUpToEpisode` 算出を含む、実効エピソード番号を必要とする全消費者がこの同一規則を消費することで、スコープ不一致による読書位置を超えたエピソード内容の漏れ（ネタバレ）を防ぐ。フォルダ `.txt` 列挙・数値プレフィクス抽出・実効エピソード番号解決・全話スコープ上限解決のプリミティブを、feature に依存しない `lib/shared/` 配下の単一の共有モジュールに集約する。

## Requirements

### Requirement: 共有のフォルダ `.txt` ファイル列挙

システムは、小説フォルダのパスから、その直下にある `.txt` ファイルの基底名を辞書順（昇順）で列挙する単一の共有関数を提供しなければならない（SHALL）。拡張子判定は大文字小文字を区別してはならない（SHALL NOT）。ディレクトリが存在しない、または読み取りに失敗した場合は空リストを返さなければならない（SHALL）。実効エピソード番号を必要とする全消費者（要約トリガ・スコープフィルタ・v5 migration）は、フォルダのファイル一覧をこの同一関数から取得しなければならない（SHALL）。

#### Scenario: 直下の .txt を辞書順で返す
- **WHEN** フォルダに `030_c.txt`、`010_a.txt`、`020_b.txt`、`note.md` が存在する状態で列挙が呼ばれる
- **THEN** `[010_a.txt, 020_b.txt, 030_c.txt]` を返す（`.md` は除外、辞書順）

#### Scenario: 存在しないディレクトリ
- **WHEN** 存在しないパスで列挙が呼ばれる
- **THEN** 空リストを返す（例外を送出しない）

#### Scenario: 拡張子は大文字小文字を区別しない
- **WHEN** フォルダに `001.TXT` が存在する状態で列挙が呼ばれる
- **THEN** `001.TXT` を結果に含める

### Requirement: 共有の数値プレフィクス抽出

システムは、ファイル名の先頭の連続する数字列を整数として抽出する単一の共有関数を提供しなければならない（SHALL）。先頭に数字列が無い場合は null を返さなければならない（SHALL）。

#### Scenario: 数値プレフィクスあり
- **WHEN** `040_chapter.txt` でプレフィクス抽出が呼ばれる
- **THEN** `40` を返す

#### Scenario: 数値プレフィクスなし
- **WHEN** `prologue.txt` でプレフィクス抽出が呼ばれる
- **THEN** null を返す

### Requirement: 現在ファイルの実効エピソード番号解決（ネタバレなし上限）

システムは、現在閲覧中のファイルに対応する「実効エピソード番号」を解決する単一の共有関数を提供しなければならない（SHALL）。解決規則は、ファイル名に数値プレフィクスがあればその値、無ければフォルダ内の `.txt` 辞書順における 1 始まりの順位とする（SHALL）。辞書順位を算出する際の母集合は、共有のフォルダ列挙関数が返すフォルダ全体のファイル一覧でなければならない（SHALL）。フォルダを列挙できない、または順位が確定できない場合は、悲観的フォールバックとして 1 を返さなければならない（SHALL）。要約トリガの「解析開始(ネタバレなし)」はこの関数の結果を `coveredUpToEpisode` として用いなければならない（SHALL）。

#### Scenario: 数値プレフィクスを持つ現在ファイル
- **WHEN** 現在ファイルが `040_chapter.txt`
- **THEN** 実効エピソード番号として `40` を返す

#### Scenario: プレフィクスなし現在ファイルは辞書順位を用いる
- **WHEN** 現在ファイルが `intro.txt`、フォルダの辞書順 `.txt` 一覧が `[intro.txt, part1.txt, part2.txt]`
- **THEN** 実効エピソード番号として `1` を返す

#### Scenario: 列挙不能時のフォールバック
- **WHEN** 現在ファイルが `intro.txt`、フォルダが列挙不能（空一覧）
- **THEN** 悲観的フォールバックとして `1` を返す

### Requirement: フォルダ全体のスコープ上限解決（ネタバレあり上限）

システムは、フォルダ内の全ファイルを包含する「全話」スコープ上限を解決する単一の共有関数を提供しなければならない（SHALL）。上限は `max(フォルダ内の最大数値プレフィクス, フォルダ内 .txt 総数)` としなければならない（SHALL）。これにより、番号付きファイルとプレフィクスなしファイルが混在するフォルダでも、プレフィクスなしファイルが総数ベースの上限により全話スコープへ含まれることを保証する（SHALL）。フォルダに `.txt` が無い場合は `1` を返さなければならない（SHALL）。要約トリガの「解析開始(ネタバレあり)」はこの関数の結果を `coveredUpToEpisode` として用いなければならない（SHALL）。

#### Scenario: 番号付きファイルのみ
- **WHEN** フォルダの最大プレフィクスが `120`、`.txt` 総数が `120`
- **THEN** 上限 `120` を返す

#### Scenario: 番号付きとプレフィクスなしの混在
- **WHEN** フォルダが `[001_ch.txt .. 040_ch.txt, prologue.txt, afterword.txt]`（最大プレフィクス `40`、総数 `42`）
- **THEN** 上限 `42` を返す（総数ベースの上限でプレフィクスなしファイルを取りこぼさない）

#### Scenario: 空フォルダ
- **WHEN** フォルダに `.txt` が存在しない
- **THEN** 上限 `1` を返す

### Requirement: スコープフィルタの辞書順位はフォルダ全体で計算

システムは、要約結果を `coveredUpToEpisode` 上限でフィルタする際、各結果ファイルの実効エピソード番号を共有の解決規則で算出し、`実効エピソード番号 <= upperBound` のものだけを残さなければならない（SHALL）。プレフィクスなしファイルの辞書順位は、検索結果の部分集合ではなく、共有のフォルダ列挙関数が返すフォルダ全体の一覧に対して計算しなければならない（SHALL）。これにより、トリガが上限を決めた際の順位とフィルタが適用する順位が常に一致し、現在の読書位置を超えた章の内容が漏れない（SHALL NOT 漏らす）。

#### Scenario: 順位がフォルダ全体で一致する
- **WHEN** フォルダの辞書順一覧が `[intro.txt, part1.txt, part2.txt, part3.txt]`、`upperBound=2`、検索結果が `[part3.txt（マッチ）, intro.txt（マッチ）]`（`part1.txt`/`part2.txt` はマッチなし）
- **THEN** `intro.txt`（順位1 <= 2）のみ残し、`part3.txt`（順位4 > 2）を除外する（結果部分集合での順位＝1,2 にはしない）

### Requirement: 全消費者が単一の共有規則を消費する

システムは、フォルダ `.txt` 列挙・数値プレフィクス抽出・実効エピソード番号解決・全話スコープ上限解決を単一の共有モジュールに集約しなければならない（SHALL）。要約トリガ解決（`analysis_runner`）・要約結果スコープフィルタ（`llm_summary_service`）・v5 migration の `coveredUpToEpisode` 算出（`novel_database`）は、これらのプリミティブを独自に再実装してはならず（SHALL NOT）、共有モジュールを消費しなければならない（SHALL）。共有モジュールは特定の feature に依存しない位置（`lib/shared/` 配下）に置き、下位レイヤから上位 feature への逆依存を生じさせてはならない（SHALL NOT）。

#### Scenario: トリガとフィルタが同一の数値プレフィクス規則を使う
- **WHEN** `030_alice.txt` に対し、トリガ解決とスコープフィルタの双方が実効エピソード番号を算出する
- **THEN** 双方とも共有関数を経由して `30` を得る（独立実装による食い違いが生じない）

#### Scenario: v5 migration が共有プリミティブを使う
- **WHEN** v5 migration が `coveredUpToEpisode` を算出する過程で数値プレフィクスと辞書順位を必要とする
- **THEN** `novel_database` 内の private 複製ではなく共有モジュールのプリミティブを消費する
