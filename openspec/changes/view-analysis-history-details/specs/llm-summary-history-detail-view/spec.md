## ADDED Requirements

### Requirement: 単語詳細ダイアログの起動

解析履歴エントリの右クリックメニューで「詳細を表示」が選択されたとき、システムはそのエントリの単語に対する read-only の詳細ダイアログを表示しなければならない（SHALL）。ダイアログは「事実」タブと「解析結果」タブの2タブ構成とし、起動直後は「事実」タブを選択状態とする。ダイアログのタイトルには対象の単語を表示する。

#### Scenario: 詳細ダイアログを開く

- **WHEN** ユーザーが履歴エントリ「アリス」の右クリックメニューで「詳細を表示」を選択する
- **THEN** タイトルに「アリス」を含む詳細ダイアログが表示される
- **AND** 「事実」タブと「解析結果」タブが表示され、「事実」タブが選択状態である

#### Scenario: ダイアログを閉じる

- **WHEN** ユーザーが詳細ダイアログの閉じる操作（閉じるボタン、ダイアログ外タップ、または Esc）を行う
- **THEN** ダイアログが閉じ、解析履歴パネルの状態は変化しない

### Requirement: 事実タブのファイル別表示

「事実」タブは、対象単語の `fact_cache` 行を `FactCacheRepository.findForWord` で取得し、ファイル（`file_name`）ごとに区切って事実テキスト（`facts`）を表示しなければならない（SHALL）。各ファイルの見出しには `file_name` と `updated_at` を表示する。表示はファイル名の昇順とする。`fact_cache` に対象単語の行が1件も存在しない場合は、空状態メッセージを表示する。

#### Scenario: 複数ファイルの事実を表示

- **WHEN** 単語「アリス」に対し `fact_cache` に `001_ch.txt` と `002_ch.txt` の2行が存在し、ユーザーが詳細ダイアログの「事実」タブを表示する
- **THEN** `001_ch.txt`・`002_ch.txt` のそれぞれの見出しの下に、対応する `facts` テキストが表示される
- **AND** ファイルはファイル名昇順で並ぶ

#### Scenario: 事実が存在しない単語

- **WHEN** 対象単語に対する `fact_cache` 行が1件も存在しない状態で「事実」タブを表示する
- **THEN** 事実が無い旨の空状態メッセージが表示される

### Requirement: 無効化された事実のグレー表示

`content_hash` がセンチネル（空文字）の `fact_cache` 行は無効化済み（次回再解析で再抽出される）であるため、「事実」タブはこれらをリストから削除せず、無効であることが視覚的に分かる表示（グレーアウト等の減衰表現と「無効」を示すバッジ／ラベル）で残さなければならない（SHALL）。

#### Scenario: 無効な事実をグレー表示で残す

- **WHEN** 単語「アリス」の `fact_cache` に、有効な行（`001_ch.txt`）と `content_hash` が空文字の無効な行（`002_ch.txt`）が存在し、「事実」タブを表示する
- **THEN** `001_ch.txt` の事実は通常表示される
- **AND** `002_ch.txt` の事実はグレーアウトされ、「無効」を示すバッジ／ラベル付きで表示される（リストから消えない）

### Requirement: 解析結果タブのスナップショット表示

「解析結果」タブは、対象単語の `word_summaries` スナップショットを `LlmSummaryRepository.findSnapshotsForWord` で取得し、既存のホバーポップアップ（`llm-summary-hover-popup`）の要約カードを流用してスナップショットの要約（`summary`）を表示しなければならない（SHALL）。複数スナップショットが存在する場合、カードのスナップショット切替 UI（◀／▶）でスナップショット間を切り替えられる。スナップショットが1件も存在しない場合は、空状態メッセージを表示する。

#### Scenario: スナップショットを要約カードで表示

- **WHEN** 単語「アリス」に複数のスナップショットが存在し、ユーザーが「解析結果」タブを表示する
- **THEN** 既存の要約カードでいずれかのスナップショットの `summary` が表示される
- **AND** スナップショット切替 UI（◀／▶）で他のスナップショットの要約に切り替えられる

#### Scenario: 解析結果が存在しない場合

- **WHEN** 対象単語に対する `word_summaries` スナップショットが存在しない状態で「解析結果」タブを表示する
- **THEN** 解析結果が無い旨の空状態メッセージが表示される

### Requirement: 詳細ダイアログは閲覧専用

詳細ダイアログは閲覧専用でなければならない（SHALL）。ダイアログ内のいかなる操作も `fact_cache` および `word_summaries` の行を変更・削除・並べ替えしてはならない（SHALL NOT）。再解析のトリガもこのダイアログからは提供しない。

#### Scenario: 操作によるデータ変更が無いこと

- **WHEN** ユーザーが詳細ダイアログ内でタブ切替・スナップショット切替・スクロールなどの操作を行い、ダイアログを閉じる
- **THEN** `fact_cache` および `word_summaries` の行は一切変更・削除・並べ替えされない
- **AND** 解析履歴パネルはこの操作を理由に再読み込みされない
