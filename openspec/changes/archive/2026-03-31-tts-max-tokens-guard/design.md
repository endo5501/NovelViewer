## Context

Qwen3-TTSモデル（特に0.6B）で、EOSトークンが検出されずmax_audio_tokens（現在4096=最大約82秒）まで生成が走り続ける問題がある。ベンチマーク対応で原因が特定され、NovelViewerでも同様の現象（5分程度の無音や意味不明な音声）が確認されている。

現状のアーキテクチャ：
- TextSegmenterは「。！？\n」でのみ文を分割
- C APIはmax_audio_tokensを外部から設定する手段がない（デフォルト4096固定）
- 1フレーム = 480サンプル / 24000Hz = 20ms → 4096フレームで約82秒

## Goals / Non-Goals

**Goals:**
- EOSミス時の最大被害時間を82秒から41秒以下に削減する
- 長い文を適切に分割してEOSミスの発生確率自体を下げる
- テキスト長に応じた動的なmax_audio_tokens制限で安全ネットを提供する

**Non-Goals:**
- EOSミスの根本原因（モデル品質）の修正
- ユーザー向けのmax_tokens設定UI
- 音声品質の改善（分割による品質変化は許容範囲内とする）

## Decisions

### 1. 二層防御アーキテクチャ

**決定**: TextSegmenterの分割改善（レイヤー1）とmax_audio_tokensの動的制限（レイヤー2）を併用する。

**理由**: レイヤー1だけでは短い文でもEOSミスが起きうる。レイヤー2だけでは長い文が途中切断される。両方を組み合わせることで、分割で入力を適切なサイズにし、動的制限で最悪ケースを制御する。

### 2. TextSegmenter分割閾値: 200文字

**決定**: 200文字を超える文は読点「、」で分割する。「、」がなければ200文字付近で強制分割する。

**理由**: 日本語1文字 ≈ 6〜10フレーム。200文字の実際の音声は約1200〜2000フレーム。max_audio_tokens上限2048に収まり、かつ一般的な小説の文は200文字以下がほとんど。

**代替案**:
- 300文字閾値: max_tokens計算が2048を超え、上限でクリップされるケースが増える
- 100文字閾値: 分割が多すぎて音声の自然さが損なわれる

### 3. max_audio_tokens計算式: `min(文字数 × 15 + 50, 2048)`

**決定**: テキスト文字数に係数15を乗じ、最小マージン50を加算、上限2048でクリップする。

**理由**:
- 係数15: ベンチマーク結果から推定。15文字で200トークンが正常(≈13.3/文字)、安全マージンを含め15
- マージン50: 極短テキスト（1〜3文字）でも最低限のトークン数を確保
- 上限2048: 最大約41秒の音声。EOSミス時でもユーザー体験として許容可能

### 4. C APIシグネチャ変更（引数追加）

**決定**: `qwen3_tts_synthesize(ctx, text, max_tokens)` のように引数を追加する。setter APIではなく直接引数にする。

**理由**: このライブラリはNovelViewerとの1対1関係で、下位互換性を維持する必要がない。呼び出しごとにmax_tokensが変わるため、引数の方がステートレスで明確。

**代替案**:
- setter API (`qwen3_tts_set_max_tokens`): 状態管理が増え、呼び出し順序の依存性が生まれる
- tts_params構造体をC APIで公開: FFIバインディングが複雑になりすぎる

### 5. C++側デフォルトmax_audio_tokensの変更

**決定**: `tts_params::max_audio_tokens` のデフォルト値を4096から2048に変更する。

**理由**: CLIツール（main.cpp）やC APIからmax_tokensを指定しない場合の安全ネット。4096は長すぎる。

## Risks / Trade-offs

- **[分割による音声品質変化]** → 読点での分割は自然な息継ぎポイントなので影響は最小限。200文字閾値で一般的なテキストではほぼ分割されない
- **[max_tokens上限による途中切断]** → 2048フレーム(41秒)を超える正当な音声はないと想定。200文字分割 + 係数15 で通常ケースは上限に達しない
- **[係数15の精度]** → ベンチマーク1件からの推定。将来的にデータを蓄積して調整可能。安全側（大きめ）に設定しているため、正常なテキストが途中切断されるリスクは低い
- **[DLLリビルド必須]** → C APIシグネチャ変更のため。Windows/macOSの両方でリビルドが必要
