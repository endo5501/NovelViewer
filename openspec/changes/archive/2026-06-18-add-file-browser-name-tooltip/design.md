## Context

ファイルブラウザ左カラムはカラム幅が狭く、フォルダタイル（`_buildDirectoryTile`）は `dir.displayName`、ファイルタイル（`_buildFileTile`）は `file.name` を `maxLines: 1, overflow: TextOverflow.ellipsis` で表示している。長い名前は省略され、フルネームを確認する手段がない。

本プロジェクトには既にホバー表示の実装が2系統存在する:
- Flutter 標準 `Tooltip`（テキストのラップのみ）
- `llm_summary/HoverPopupHost` による自前 `Overlay` 方式（Provider/Anchor/Widget 一式）

## Goals / Non-Goals

**Goals:**
- フォルダタイル・ファイルタイルのホバー時に、省略されていないフルネームを表示する。
- レイアウト・既存の選択装飾・タップ挙動を変更しない。

**Non-Goals:**
- ファイル名以外の追加情報（フォルダ名 `name` の併記、進捗、メタデータ等）の表示。
- 名前が省略されている時だけ表示する条件分岐（常時表示とする）。
- 自前 Overlay ポップアップの導入。

## Decisions

### 決定1: Flutter 標準 `Tooltip` を使用する

タイトル/ファイル名の `Text` を `Tooltip(message: ..., child: Text(...))` でラップする。

- **理由**: デスクトップではホバーで自動表示され、遅延・テーマ・アクセシビリティが標準で備わる。実装量が最小で、要件（フルネームを出すだけ）に対して過不足がない。
- **代替案**: `llm_summary` の自前 `Overlay` 方式。即時表示や位置自由・リッチな中身が可能だが、単語意味表示・表示モード追従という別要件のための重装備であり、本件にはオーバーキル。採用しない。

### 決定2: ツールチップは `Text` の直上に置く

ファイルタイルは選択中に `Container`（装飾）+ `Material` でラップされる分岐がある。`Tooltip` は `ListTile` の `title` に渡す `Text` の直上に配置し、選択状態の有無に関わらず同じ位置に置く。

- **理由**: 選択装飾のラップ分岐から独立し、選択/非選択どちらでも一貫してホバー表示が働く。

### 決定3: ツールチップ内容

- フォルダタイル: `dir.displayName`
- ファイルタイル: `file.name`

## Risks / Trade-offs

- [省略されていない名前でもホバーで表示される] → 仕様として許容（カラム幅が狭く省略されない場面はまれ、という判断）。冗長だが害はない。
- [`Tooltip` のデフォルト表示遅延（待機時間）] → 標準挙動のまま採用。必要なら `waitDuration` で調整可能だが、本件では既定値で十分。
