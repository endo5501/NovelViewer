## Context

`TextSegmenter`（`lib/features/tts/data/text_segmenter.dart`）はTTS再生のためにテキストを文単位へ分割する。現状は日本語専用で、以下のみを認識する。

- 文末記号: `_sentenceEnders = {'。', '！', '？'}`（全角のみ）
- 閉じ括弧: `_closingBrackets = {'」', '』', '）'}`（全角のみ）
- 長文分割の読点: `_findSplitPosition` 内の `、`（全角のみ）

このため英文は文末で区切られず1段落=1セグメントになり、200文字超で `_splitByLength` → `_findSplitPosition` が起動するが、`、`もスペースも探さないため**200文字ちょうどで単語の途中を強制分割**する。これが報告された "scales｜with usage"・"pr｜ototype" の正体。

制約:
- `offset` / `length` は表示テキスト座標でUIハイライトに使われるため、分割位置変更時も整合を保つ必要がある。
- 既存の日本語テストおよび挙動を一切壊さない（後方互換）。
- 200文字上限は `tts-text-length-guard` の暴走防止ガードであり維持する。

## Goals / Non-Goals

**Goals:**
- 英文を `.` `!` `?` の文末、`,` の読点、語境界（スペース）で自然に分割する。
- 単語の途中での分割を解消する。
- 日本語のみのテキストでは出力を不変に保つ。

**Non-Goals:**
- 完全な自然言語的文境界判定（略語 `Mr.`・`e.g.`、頭字語 `U.S.` の厳密処理）は行わない。ヒューリスティックで実用十分とする。
- 言語自動判定の導入はしない（記号ベースで日英を同時に扱う）。
- 200文字上限値そのものの変更はしない。

## Decisions

### 1. 半角文末記号は「直後が境界」のときのみ文末とみなす
`.` `!` `?` を `_sentenceEnders` 相当として扱うが、半角 `.` は小数点・略語にも現れる。誤分割を避けるため、半角文末記号は**直後の文字が空白・文字列終端・閉じ括弧のいずれか**のときのみ文末と判定する。

- `3.14` → `.` の直後が `1`（非境界）なので分割しない。
- `prompt. The` → `.` の直後が空白なので分割する。
- 全角 `。！？` は従来どおり無条件で文末（日本語は分かち書きしないため）。

**代替案**: 半角も無条件で文末とする → `3.14` や URL が壊れるため却下。

### 2. 半角閉じ括弧/引用符を閉じ括弧集合に追加
`"` `)` を文末直後にまとめる閉じ括弧として追加（`He said "Run." ` の `."` を1セグメント側へ）。`'`（アポストロフィ）は所有格・短縮形と衝突するため**追加しない**。

### 3. 長文分割は「読点 → 空白 → 強制」の優先順位
`_findSplitPosition` を次の優先順位に変更する。

1. 先頭200文字以内の**最後の読点**（`、` または `,`）→ 読点を第1セグメントに含めて分割（既存の全角挙動と同一）。
2. 読点が無ければ、200文字以内の**最後の空白**（単語境界）→ 空白の手前で分割し、残りは `trimLeft` で先頭空白を除去。
3. 空白も無ければ従来どおり**200文字で強制分割**。

これにより英文は読点優先・なければ語境界で切れ、単語の途中分割が起きるのは「200文字以内に空白が1つも無い」病的ケースのみになる。

**代替案**: 常にスペース分割を最優先 → 読点での区切りという要望に反するため却下。

### 4. 空白判定は単一文字の `trim().isEmpty` で行う
半角スペース・タブ・全角スペース等を一括で空白とみなせるため、文字種を列挙せず `char.trim().isEmpty` を用いる。改行は `_splitTextBySentence` の既存ループで先に処理されるため二重処理にならない。

## Risks / Trade-offs

- [略語・頭字語で過分割（`Mr. Smith` → `Mr.` で切れる）] → TTS用途では文が短く切れても読み上げ品質への実害は軽微。Non-Goalとして許容し、必要なら将来ストップワード辞書で対応。
- [`offset`/`length` のズレ] → 英文の空白/読点分割では `_splitByLength` の `trimLeft()` が落とす空白分だけ後続セグメントのオフセットが累積的にずれる（UIハイライトの位置ずれ）。プレーンテキスト（`totalTextLen == totalDisplayLen`）では落とした空白文字数 `droppedLen` を表示カーソルに加算してオフセットを補正する。ルビ置換時は従来の比例推定で近似する。`offsets stay aligned ...` テストで回帰確認する。
- [既存日本語挙動の破壊] → 追加判定は半角文字に対してのみ発火し、全角経路は条件分岐を増やさない。既存テスト全通過を完了条件とする。
