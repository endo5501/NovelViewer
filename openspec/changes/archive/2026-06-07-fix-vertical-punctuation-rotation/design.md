## Context

縦書きレンダリング（`vertical_text_page.dart` / `vertical_ruby_text_widget.dart`）は、文字ごとに固定幅 `SizedBox` を割り当て、`mapToVerticalChar()` で「横書き専用文字 → 縦書き用Unicode字形（presentation form）」へ**置換**することで縦書き化している。`Transform`/`RotatedBox` による物理回転は現状一切使われていない。

この置換方式は、フォントが縦書き字形グリフを持つ文字（括弧 `﹁ ﹂ ︵ ︶` 等）では機能する。しかし以下では破綻している:

- **クオート類**（`"` `＂` `"` `"` / `'` `＇` `'` `'` / `` ` `` `｀`）: そもそもUnicodeに縦書き用presentation formが存在せず、置換先が無い。セル右上に小さく浮いた不自然な表示になる（`tmp/ダブルクオーテーション.png`）。
- **コロン・セミコロン**（`:` `：` `;` `；`）: 既に `︓`(U+FE13)/`︔`(U+FE14) へ置換済みだが、対象フォントが当該CJK互換形を回転グリフとして描画できず、点が縦に並んだまま（`tmp/コロン.png`）。

`〝〟`（U+301D/U+301F）への置換も検証したが、同様にセル右上に浮き不自然だった（`tmp/ダブルクオート置換テスト.png`）。

## Goals / Non-Goals

**Goals:**
- 上記の対象文字を、フォント依存しない方法で縦書きとして自然に見せる。
- 置換方式と物理回転方式を、文字種ごとに使い分けられる仕組みを導入する。
- 文字を置換しない（＝文字数・テキストオフセットを変えない）ことで、検索ハイライト/TTSハイライトのオフセット整合を維持する。

**Non-Goals:**
- 既存の括弧・句読点・ダッシュ等の置換挙動の変更（現状維持）。
- ダブルクオートを `「」『』` 等の和文約物へ意味変換すること（開き/閉じ判別が必要で誤変換リスクがあるため対象外）。
- 設定UIによる回転ON/OFF切り替えの追加。

## Decisions

### 決定1: 物理回転（Transform.rotate）方式を採用

対象文字を `Transform.rotate(angle: π/2)`（時計回り90°）で回転させて描画する。`mapToVerticalChar` による置換は通さず、原文字をそのまま回転する。

- **なぜ Transform.rotate か（RotatedBox ではなく）**: `RotatedBox(quarterTurns: 1)` は回転後の寸法（子の幅/高さを入れ替えた寸法）でレイアウトされる。半角の `:` `;` やクオートは横幅が小さいため、回転後のセル**高さ**が大幅に縮み、固定幅 `SizedBox`（幅のみ固定）では通常文字1字分の高さを消費しなくなる。縦組みのページング（`TextPainter('あ')` の固定 `charHeight` 前提で列分割）・ハイライト背景・マーク線・列末位置が実描画とずれる回帰を生む。`Transform.rotate` は描画のみを回転しレイアウト寸法は子（＝通常の `Text`）のまま変わらないため、セル寸法は通常文字と同一に保たれる。中央寄せは `alignment: Alignment.center`（既定）で担保する。
- **なぜ時計回り90°か**: 縦書きにおけるラテン約物の標準的な回転方向（UAX#50 の "R" 指向）に倣う。Flutterの座標系では正の角度が時計回りのため `angle: π/2`。コロンの点は左右対称で方向不問だが、クオート/アポストロフィは方向で見え方が変わるため統一方向とし、実機で目視確認する。

**代替案1**: `RotatedBox(quarterTurns: 1)`。→ 却下。上記のとおりセル高が縮み行リズムが崩れる。
**代替案2**: 縦書き用Unicode字形への置換（`〝〟` 等）。→ 却下。クオートのシングル/バッククォートには適切な字形が無く、コロンで実証済みの「フォントが互換形を回転描画できない」リスクを再び負う。

### 決定2: 回転対象集合を新設し、コロン・セミコロンをマップから移管

`vertical_char_map.dart` に回転対象文字の集合（例: `verticalRotateChars`）を新設する。

回転対象:
- ダブルクオート: `"`(U+0022) `＂`(U+FF02) `"`(U+201C) `"`(U+201D)
- シングルクオート/アポストロフィ: `'`(U+0027) `＇`(U+FF07) `'`(U+2018) `'`(U+2019)
- バッククォート: `` ` ``(U+0060) `｀`(U+FF40)
- コロン: `:`(U+003A) `：`(U+FF1A)
- セミコロン: `;`(U+003B) `；`(U+FF1B)

併せて `verticalCharMap` から `:` `：` `;` `；` のエントリ（`︓`/`︔` への置換）を**削除**する。これらは回転側で扱う。

### 決定3: レンダリング分岐は描画箇所で判定

`_buildCharWidget`（`vertical_text_page.dart`）で、文字が回転対象集合に含まれる場合は `RotatedBox` で包んだ `Text(原文字)` を、そうでなければ従来どおり `Text(mapToVerticalChar(原文字))` を、いずれも固定幅 `SizedBox` 内に配置する。

判定ヘルパ（例: `shouldRotateVertical(String char) -> bool`）を `vertical_char_map.dart` に置き、データ層に文字分類ロジックを集約する（既存 `mapToVerticalChar` と同居）。

### 決定4: ルビ経路の扱い

`vertical_ruby_text_widget.dart` もベース文字に `mapToVerticalChar` を適用している。ルビのベース/ルビ文字に回転対象約物が現れた場合も同じ回転規則を適用し、表示の一貫性を保つ。実装の重複を避けるため、回転判定・回転描画は共通ヘルパ経由とする。

## Risks / Trade-offs

- [回転方向がクオートで不自然に見える可能性] → 実機（Windows/macOS）で目視確認し、必要なら回転角を調整。tasks に確認手順を含める。
- [ダブルクオート回転後が「横棒2本の縦積み」に見える] → 置換案より自然と合意済み（`tmp/ダブルクオート置換テスト.png` 比較）。許容する。
- [セル高の縮みによる行リズム崩れ] → `RotatedBox` ではなく `Transform.rotate` を用い、レイアウト寸法を通常文字と同一に保つ。「回転セル高 == 通常セル高」をウィジェットテストで回帰確認する。
- [ヒットテスト/選択範囲のセル矩形ずれ] → `Transform.rotate` はレイアウト寸法を変えず固定幅 `SizedBox` 内に収まるため、セル矩形（GlobalKey 収集）は従来同様に取得できる。選択・検索ハイライトのテストで回帰確認する。
- [ルビ経路への波及] → 回転判定を共通ヘルパに集約し、本体・ルビで挙動を一致させることで二重実装を防ぐ。
