## Context

現在の `parseEpisode()` は、HTMLの `<p>` タグを `\n\n` で結合し、空の `<p>`（`<br>` のみ含む）は `_blockToText()` の `trim()` で空文字になり `isNotEmpty` チェックでスキップされる。

実際のなろうHTMLの構造:
```html
<p id="L1">文1</p>           ← 通常行
<p id="L2">文2</p>           ← 通常行
<p id="L3"><br /></p>         ← 場面転換の空行
<p id="L4">文3</p>           ← 通常行
```

Web画面では `<p>` 間は1行分の間隔、空 `<p>` は空行として表示される。

## Goals / Non-Goals

**Goals:**
- ダウンロードしたテキストの行間をWeb画面の表示と一致させる
- 場面転換などの意図的な空行を保持する

**Non-Goals:**
- テキストビューアー側の表示ロジック変更
- 既にダウンロード済みファイルの自動修正
- `_blockToText()` 内の `<br>` → `\n` 変換の変更（これは正しく動作している）

## Decisions

### 1. `<p>` 間のセパレータを `\n` に変更

`<p>` タグはWeb画面上で1行分の改行として表示される。テキストファイルでも同様に `\n`（1改行）で結合する。

代替案: セパレータをなくす → `<br>` がない `<p>` 間の改行が消えてしまう。不採用。

### 2. 空の `<p>` を空行として保持

`_blockToText()` が空文字を返した場合（`<br>` のみの `<p>` など）、スキップせずにセパレータとして `\n` を追加書き込みする。これにより、空 `<p>` 1つにつき空行1つが出力される。

具体的な変換結果:
```
<p>文1</p><p><br/></p><p>文2</p>
  → "文1\n\n文2"  （空<p>が\nを生成 + 通常セパレータ\n = \n\n）

<p>文1</p><p><br/></p><p><br/></p><p>文2</p>
  → "文1\n\n\n文2"  （空<p>×2 = 空行2つ）
```

## Risks / Trade-offs

- [カクヨムのHTML構造が異なる可能性] → カクヨムの実際のHTMLも確認して同じパターンか検証する
- [既存テキストとの不一致] → 再ダウンロードで対応。既存ファイルは変更しない
