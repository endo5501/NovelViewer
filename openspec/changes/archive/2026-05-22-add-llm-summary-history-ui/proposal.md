## Why

LLM単語解析(ネタバレあり/なし)のキャッシュは存在するが、ユーザが過去に何を調べたかを思い出す手段がない。同じ単語を再度調べてしまったり、せっかく解析したのに存在を忘れて活用できないという課題がある。本文中に「過去に調べた単語」が視認できるマークを表示し、また左カラムに解析履歴を一覧/管理する専用タブを設けることで、解析結果を読書体験に組み込む。

## What Changes

- 本文中のキャッシュ済み単語に視覚マークを描画する(横書き=下線、縦書き=傍線)
  - ネタバレなしキャッシュ = 点線 / ネタバレありキャッシュ = 実線
  - 両方ある場合は実線優先
  - マッチング: 最長一致 + 最小長2文字(短すぎる単語は誤マーク回避のため対象外)
  - ルビ付きテキストではベース文字側にのみマークを付ける
  - 既存ハイライト(検索=黄背景、TTS=緑背景)とは独立した視覚チャネル(下線/傍線)を使用
- 左カラムに3つ目のタブ「解析履歴」を追加(現行: ファイル / ブックマーク → 改修後: ファイル / ブックマーク / 解析履歴)
- 解析履歴タブで現在開いている作品(フォルダ)内のキャッシュ済み単語を更新日時降順で一覧表示
- 各エントリは単語、タイプ(なし/あり/両)、要約のプレビュー、更新日時を表示
- エントリのクリックで、そのキャッシュの source_file を開きテキスト内の最初の出現箇所へジャンプ
- エントリの右クリックで「削除」コンテキストメニューを表示
- スキーマ拡張: ネタバレあり(spoiler)エントリにも `source_file` を保存する(ジャンプ先解決のため)。既存の spoiler 行は `source_file=NULL` のままで、その場合はジャンプ動作の対象外とする

## Capabilities

### New Capabilities
- `llm-summary-history-ui`: 解析履歴タブのUI(エントリ一覧、削除、テキスト内ジャンプ)

### Modified Capabilities
- `llm-summary`: 本文中の解析履歴マーク(下線/傍線)の描画要件を追加
- `llm-summary-cache`: spoiler エントリにも `source_file` を保存する要件、最小長2文字のフィルタ要件を追加
- `bookmark-ui`: 左カラムタブを2つから3つ(ファイル/ブックマーク/解析履歴)に拡張

## Impact

- 影響コード:
  - `lib/features/llm_summary/` 配下 (リポジトリ・サービス・UI)
  - `lib/features/text_viewer/` 配下 (本文描画にマーク span を組み込む)
  - `lib/features/bookmark/presentation/left_column_panel.dart` (タブ追加)
  - 新規 `lib/features/llm_summary/presentation/llm_summary_history_panel.dart`
- DB スキーマ: `word_summaries` テーブルの spoiler 行への `source_file` 書き込み開始(カラム自体は既存。マイグレーション不要)
- 依存: 既存の Riverpod プロバイダ、SQLite (sqflite_common_ffi)
- パフォーマンス: 1ファイル数万字 × 数百単語規模を想定。トライ木または最長一致対応の正規表現で本文1パスでマーク位置を計算
- 後方互換: 既存 spoiler エントリ(`source_file=NULL`)は履歴一覧に表示されるが、ジャンプ非対応として扱う
