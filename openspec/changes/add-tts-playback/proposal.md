## Why

NovelViewer は小説を閲覧するアプリケーションだが、現在テキストの読み上げ機能がない。長時間の読書で目を休めたいときや、ながら読みしたいときに音声で小説を楽しめるようにするため、ローカル TTS エンジン（qwen3-tts.cpp）による読み上げ機能を追加する。

## What Changes

- qwen3-tts.cpp を git サブモジュールとして追加し、共有ライブラリ（.dylib/.dll）としてビルドする
- C API ラッパーを作成し、dart:ffi で Flutter から呼び出せるようにする
- テキストビューアに再生/停止ボタンを追加する
- 再生開始時、選択テキスト位置から開始する（選択がなければ表示先頭から）
- テキストを文単位（句点・改行）で分割し、1文ずつ音声を生成・再生する
- 読み上げ中の文をハイライト表示する（読み上げ終了後にハイライトを解除）
- 読み上げの進行に合わせて自動ページ送りを行う
- 再生停止は、停止ボタン押下またはユーザーのページ操作で行う
- 設定画面をタブ化し、TTS 設定用タブを追加する
- TTS 設定タブで GGUF モデルディレクトリのパスとクローン元 WAV ファイルのパスを設定できるようにする
- 対象プラットフォームは macOS と Windows のみ
- モデル変換・配置はユーザーが手動で行う前提

## Capabilities

### New Capabilities
- `tts-native-engine`: qwen3-tts.cpp のサブモジュール管理、共有ライブラリビルド、C API ラッパー、dart:ffi バインディング
- `tts-playback`: テキスト分割（文単位）、音声生成キュー、再生制御（開始・停止）、テキストハイライト同期、自動ページ送り、再生開始位置の決定
- `tts-settings`: 設定画面のタブ化、TTS 設定タブ（モデルディレクトリパス、WAV ファイルパス）、設定の永続化

### Modified Capabilities
- `text-viewer`: 再生/停止ボタンの追加、TTS 読み上げ中のハイライト表示対応、TTS からのページ送り要求への対応

## Impact

- **依存関係**: qwen3-tts.cpp（git サブモジュール）、GGML ライブラリ、音声再生用 Flutter パッケージ（just_audio 等）を追加
- **ビルドシステム**: macOS/Windows のネイティブビルド設定に共有ライブラリのビルドステップを追加
- **ネイティブコード**: 初の FFI 統合。C API ラッパーの追加と、プラットフォームごとのライブラリロードパス管理が必要
- **設定画面**: 既存の設定ダイアログをタブ形式に変更（既存設定は「一般」タブに移動）
- **テキストビューア**: 横書き・縦書き両モードで再生 UI とハイライト機能を追加
- **ファイルサイズ**: GGUF モデルファイル（数百 MB〜数 GB）はユーザーが手動配置するため、アプリバンドルには含まない
